# คู่มือแพ็กเกจหลักในโครงการ (Project Packages & Libraries)

เอกสารนี้ระบุรายละเอียด หน้าที่การทำงาน และการอัปเดตสถานะของแพ็กเกจ/ไลบรารีย่อยทั้งหมดในระบบ **LowcodeProject** เพื่อเป็นข้อมูลอ้างอิงในการพัฒนา

---

## 📦 1. แพ็กเกจหลักของระบบ (Core & Framework Packages)

### 📌 @refinedev/core (`packages/core`)
* **มันคืออะไร?**: หัวใจหลักของระบบ Refine (The brain of the framework)
* **ทำงานอย่างไร?**: 
  * จัดการกระบวนการเรียกใช้ API (Data Providers), จัดการระบบสมาชิก (Auth Providers), และจัดการเส้นทาง (Routing) 
  * ทำหน้าที่ประมวลผล Logic ต่าง ๆ หลังบ้านของหน้าจอ โดยส่งข้อมูลให้ฝั่ง UI ไปแสดงผล
* **สร้างวันแรก**: 2022
* **อัปเดตล่าสุด**: 2026-07-10 (โดย Tichakorn Khunyai)
* **สถานะความสำคัญ**: 🔴 **ห้ามลบเด็ดขาด (Required)**

### 📌 @refinedev/antd (`packages/antd`)
* **มันคืออะไร?**: ส่วนเชื่อมต่อจัดการหน้าจอการแสดงผลด้วย **Ant Design** (UI/UX Layer)
* **ทำงานอย่างไร?**:
  * แปลง Logic ของ Refine Core ออกมาเป็นหน้าจอตารางข้อมูล (Table), ฟอร์มนำเข้าข้อมูล (Form), และหน้าต่างจัดการระบบ CRUD สำเร็จรูปที่สวยงามโดยอัตโนมัติ
* **สร้างวันแรก**: 2022
* **อัปเดตล่าสุด**: 2026-07-10 (โดย Tichakorn Khunyai)
* **สถานะความสำคัญ**: 🔴 **ห้ามลบเด็ดขาด (Required)**

---

## 🔗 2. แพ็กเกจสำหรับเชื่อมต่อระบบภายนอก (Data & API Integrations)

### 📌 @refinedev/supabase (`packages/supabase`)
* **มันคืออะไร?**: ตัวเชื่อมต่อ (Data Provider) ระหว่างระบบ Refine และฐานข้อมูล **Supabase**
* **ทำงานอย่างไร?**:
  * ช่วยให้ระบบ Refine สามารถอ่าน เขียน อัปเดต และลบข้อมูล (CRUD) รวมทั้งจัดการสมาชิกและระบบไฟล์ใน Supabase (Local Docker) ได้ผ่านโค้ดชุดคำสั่งสำเร็จรูปเพียงไม่กี่บรรทัด
* **สร้างวันแรก**: 2022
* **อัปเดตล่าสุด**: 2026-07-10 (โดย Tichakorn Khunyai)
* **สถานะความสำคัญ**: 🔴 **ห้ามลบเด็ดขาด (ใช้เชื่อมฐานข้อมูลภายใน)**

### 📌 @refinedev/simple-rest (`packages/simple-rest`)
* **มันคืออะไร?**: ตัวเชื่อมต่อข้อมูลสำหรับ REST API ทั่วไป
* **ทำงานอย่างไร?**:
  * ใช้ดึงข้อมูลจาก Endpoint URL ทั่วไปที่ส่งข้อมูลกลับมาเป็น JSON (เช่น Mock API หรือระบบภายนอกที่ต้องการคุยด้วย)
* **สร้างวันแรก**: 2022
* **อัปเดตล่าสุด**: 2026-07-10 (โดย Tichakorn Khunyai)
* **สถานะความสำคัญ**: 🟡 **แนะนำให้เก็บไว้ใช้ในการต่อ API ภายนอก**

## 🧠 3. แพ็กเกจช่วยเหลือและเครื่องมือพัฒนา (Utilities & DevTools)

### 📌 @refinedev/inferencer (`packages/inferencer`)
* **มันคืออะไร?**: เครื่องมือ **Auto Code Generator** (ตัวช่วยสร้างโค้ดอัตโนมัติ)
* **ทำงานอย่างไร?**:
  * เมื่อระบุตารางข้อมูลปลายทาง Inferencer จะวิเคราะห์ข้อมูลและสร้างหน้าสร้าง/แก้ไข/ดู/แสดงตาราง ให้คุณเห็นบนหน้าจอทันที เพื่อให้คุณกดคัดลอกโค้ดมาใช้เขียนต่อ
* **สร้างวันแรก**: 2022
* **อัปเดตล่าสุด**: 2026-07-10 (โดย Tichakorn Khunyai)
* **สถานะความสำคัญ**: 🟢 **สะดวกมากสำหรับงานสไตล์ Low-code**

### 📌 @refinedev/devtools (`packages/devtools`)
* **มันคืออะไร?**: เครื่องมือ debug ตัวช่วยตรวจสอบการทำงานของโปรเจกต์
* **ทำงานอย่างไร?**:
  * แสดงข้อมูลสถิติของแอปที่กำลังเปิดรัน เช่น ประสิทธิภาพการดึงข้อมูล โครงสร้างเมนู และประวัติการคุยกับ API เพื่อการตรวจสอบปัญหาแบบเจาะลึก
* **สร้างวันแรก**: 2023
* **อัปเดตล่าสุด**: 2026-07-10 (โดย Tichakorn Khunyai)
* **สถานะความสำคัญ**: 🟢 **ใช้ในหน้าจอพัฒนา (Development Only)**
