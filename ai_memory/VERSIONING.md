# คู่มือการอัปเดตเวอร์ชันโครงการ (Project Versioning Guide)

เอกสารนี้อธิบายวิธีการจัดการและอัปเดตเลขเวอร์ชัน (Versioning) ในโครงการ **LowcodeProject** เพื่อความเป็นระเบียบเรียบร้อยและได้มาตรฐานวิศวกรรมซอฟต์แวร์สากล

---

## 📌 มาตรฐานการตั้งเลขเวอร์ชัน (Semantic Versioning - SemVer)

เราใช้ระบบเลขเวอร์ชันแบบ `MAJOR.MINOR.PATCH` (เช่น `1.0.0`):
1. **PATCH (เลขตัวท้าย: 1.0.x)**: ขยับเมื่อมี **การแก้ไขบั๊ก (Bug Fixes)** หรืออัปเดตเล็ก ๆ น้อย ๆ ที่ไม่ส่งผลกระทบต่อการทำงานเดิม
2. **MINOR (เลขตัวกลาง: 1.x.0)**: ขยับเมื่อมี **การเพิ่มฟีเจอร์ใหม่ (New Features)** ที่ยังสามารถทำงานร่วมกับโค้ดเดิมได้
3. **MAJOR (เลขตัวแรก: x.0.0)**: ขยับเมื่อมี **การเปลี่ยนระบบครั้งใหญ่ (Breaking Changes)** โค้ดรุ่นเก่าจะไม่สามารถทำงานร่วมกับเวอร์ชันนี้ได้

---

## 🛠️ วิธีการอัปเดตเวอร์ชันในโครงการ

คุณสามารถเลือกอัปเดตเวอร์ชันได้ 3 รูปแบบตามความเหมาะสม:

### วิธีที่ 1: ให้ AI จัดการอัปเดตให้แบบอัตโนมัติ (แนะนำ)
เมื่อ AI ทำงานตามกิ่งแยก (เช่น `feature/`) เสร็จสมบูรณ์แล้ว ก่อนจะทำ Pull Request คุณสามารถสั่งให้ AI อัปเดตเวอร์ชันได้โดยใช้คำสั่งนี้:
> *"ช่วยขยับเลขเวอร์ชันโปรเจกต์เป็นระดับ [Patch / Minor / Major] และอัปเดตไฟล์ package.json กับ CHANGELOG.md ให้หน่อย"*
* AI จะทำการแก้ไขเลขเวอร์ชันใน `package.json` และนำประวัติงานไปเขียนบันทึกเพิ่มเติมใน `CHANGELOG.md` ให้ทันที

### วิธีที่ 2: ใช้ระบบอัตโนมัติของโครงการ (pnpm Changesets)
เนื่องจากระบบนี้เป็น Monorepo เราจึงใช้งานเครื่องมือ **Changesets** ในการดูแลเวอร์ชันของแพ็กเกจย่อย:
1. รันคำสั่งใน Terminal เพื่อสร้างบันทึกการเปลี่ยนแปลง:
   ```bash
   pnpm changeset
   ```
2. ตอบคำถามในหน้าจอเพื่อเลือกระดับการขยับเวอร์ชัน และพิมพ์รายละเอียดสรุปการแก้ไขสั้น ๆ (ไฟล์บันทึกชั่วคราวจะถูกสร้างในโฟลเดอร์ `.changeset/`)
3. เมื่อต้องการนำบันทึกเหล่านั้นไปอัปเดตเลขเวอร์ชันลงไฟล์จริง ให้รันคำสั่ง:
   ```bash
   pnpm changeset version
   ```
   *(ระบบจะขยับเวอร์ชันใน package.json และเขียน Changelog ให้โดยอัตโนมัติ)*

### วิธีที่ 3: ใช้คำสั่งมาตรฐานในการขยับเวอร์ชัน (Manual)
หากต้องการสั่งอัปเดตเลขใน `package.json` ของโครงการหลักตรง ๆ สามารถพิมพ์สั่งการในโฟลเดอร์หลักผ่าน Terminal:
* **แก้ไขบั๊กย่อย**: `npm version patch`
* **เพิ่มฟีเจอร์ใหม่**: `npm version minor`
* **เปลี่ยนระบบใหญ่**: `npm version major`
