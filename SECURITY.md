# นโยบายความปลอดภัยและการแจ้งเตือน (Security Policy)

เอกสารนี้ระบุแนวทางปฏิบัติในการรักษาความปลอดภัยของระบบและการรายงานช่องโหว่ความปลอดภัยภายในทีมพัฒนา **LowcodeProject**

---

## 🔒 การรายงานช่องโหว่ความปลอดภัย (Reporting a Vulnerability)

หากพบช่องโหว่ความปลอดภัย ระบบทำงานผิดปกติ หรือการรั่วไหลของข้อมูลในระบบ **ห้ามเปิดเผยข้อมูลนี้ในพื้นที่สาธารณะหรือสร้าง Issue ทั่วไปบน GitHub/GitLab** 

โปรดรายงานปัญหาโดยตรงผ่านช่องทางภายในดังนี้:
* **ผู้ดูแลระบบหลัก (System Administrator)**: แจ้งเตือนผ่านระบบตั๋วงานภายใน หรือติดต่ออีเมลผู้ดูแลระบบของโครงการ
* **หัวหน้าทีมพัฒนา (Lead Developer)**: แจ้งข้อมูลโดยละเอียดพร้อมขั้นตอนการทดสอบ (Proof of Concept) เพื่อให้ทีมเข้าสืบสวนและปิดช่องโหว่โดยเร็วที่สุด

---

## 🛡️ แนวทางปฏิบัติเพื่อความปลอดภัยของข้อมูล (Security Best Practices)

ทีมงานทุกคนต้องปฏิบัติตามกฎเพื่อป้องกันข้อมูลระบบรั่วไหล:

### 1. การจัดการรหัสผ่านและ API Key (Credential Management)
* **ห้ามบันทึก API Key ของ Gemini** หรือข้อมูลสำคัญอื่น ๆ ลงในระบบ Git เด็ดขาด
* ทุกเครื่องพัฒนาต้องใช้ไฟล์ `.env.local` หรือไฟล์กำหนดค่าที่ไม่ถูกติดตามโดย Git ในการจัดเก็บความลับของระบบ

### 2. การควบคุมการเข้าถึงฐานข้อมูล (Database Access)
* การพัฒนา Supabase ในเครื่องพัฒนาส่วนบุคคลจะต้องทำผ่าน Docker เท่านั้น
* ห้ามเปิดพอร์ตฐานข้อมูลภายในออกสู่สาธารณะหากไม่มีการตั้งระบบความปลอดภัยขั้นสูง (เช่น VPN) ครอบไว้

### 3. การประมวลผลเอกสารกับ AI (Gemini AI Processing)
* ข้อมูลไฟล์ส่วนตัวของผู้ใช้จะถูกอัปโหลดและจัดเก็บไว้ภายในระบบ Supabase Storage เครื่อง Local เท่านั้น
* เฉพาะเนื้อหาข้อความที่ต้องการส่งไปให้ AI ประมวลผลถอดข้อมูลเท่านั้นที่จะส่งผ่าน API ออกไปหา Google Gemini
